# 收尾调研摘要

## 可直接落到本项目的规则

- 长篇结尾最怕的是“已经落锤了，还在继续落锤”。高潮之后如果继续用同构场景、同构验证、同构说明反复证明同一个结论，读者对结尾的记忆会明显变钝。
- 好结尾通常不是把所有信息讲得更全，而是把前文已经建立的冲突、人物变化、代价和意象集中兑现，然后停在一个足够清晰的最后画面上。
- 余味来自“主线义务已完成后留下的一点空白”，不是靠关键问题没交代、关键责任没落地、关键情感没回应来制造。
- 连载尾声不宜拖成说明会。制度、世界观、规则变化可以出现，但只能作为最后一击的支撑材料，不能反过来压死人物和画面。
- 更稳的终局结构通常是：
  1. 一次真正不可逆的最终落锤。
  2. 一次普通层面的现实验证，证明新状态不是只在主角层面成立。
  3. 一个极短的余韵画面，让读者停在结尾而不是停在说明上。
- “意料之外但情理之中”依然是高评价结尾的硬标准。结尾不能靠突然升级反派、突然追加黑幕、突然加大设定来硬抬火力。

## 本项目的直接改法

- `ending_quality` 评估时，不再只看“有没有写完”，而是额外扣查：
  - 是否在高潮后重复落锤。
  - 是否把尾声写成制度说明会、条款会、总结会。
  - 是否靠拖尾来伪造余味。
  - 最后一屏是否可停留。
- `ending_guidance` 生成时，明确要求：
  - 最后阶段最多保留一次现实验证。
  - 不允许把后续运行机制讲成长尾。
  - 必须指出“最后一屏应该让读者看到什么、感觉到什么”。
